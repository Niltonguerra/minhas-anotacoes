# Explicação dos tópicos:

Design de código são uma serie de boas práticas para se considerar durante a escrita do código

| Conceito                            | Por que importa                                                                                        |
| ----------------------------------- | ------------------------------------------------------------------------------------------------------ |
| **KISS – Keep It Simple, Stupid**   | Mantém o código simples e direto, facilitando entendimento, manutenção e redução de erros.             |
| **DRY – Don’t Repeat Yourself**     | Evita duplicação de código, facilitando mudanças e evitando inconsistências.                           |
| **SOLID (Princípios da POO)**       | Conjunto de princípios para criar código modular, flexível e fácil de manter.                          |
| **YAGNI – You Ain’t Gonna Need It** | Evita implementar funcionalidades desnecessárias, focando no que realmente importa agora.              |
| **Code Smells e Refatoração**       | Identifica problemas no código e melhora sua estrutura sem alterar o comportamento.                    |
| **Nomeação e Semântica**            | Usa nomes claros e significativos, tornando o código mais legível e fácil de entender.                 |
| **Coesão e Acoplamento**            | Promove alta coesão (partes relacionadas juntas) e baixo acoplamento (independência entre partes).     |
| **Testabilidade**                   | Facilita a criação de testes automatizados para garantir qualidade e prevenir bugs.                    |
| **GRASP**                           | Ajuda a definir responsabilidades claras entre objetos e classes, melhorando organização.              |
| **Design Patterns**                 | Aplica soluções comprovadas para problemas comuns, aumentando reutilização e compreensão.              |
| **Lei de Demeter**                  | Reduz o acoplamento ao limitar as interações entre objetos, aumentando o encapsulamento.               |
| **Imutabilidade**                   | Prefere objetos que não mudam estado, tornando o código mais previsível e seguro.                      |
| **Separação de preocupações**       | Divide o sistema em partes com responsabilidades distintas para facilitar manutenção e evolução.       |
| **Camadas e arquitetura**           | Estrutura o código em camadas ou módulos para melhor organização, escalabilidade e teste.              |
| **Documentação**                    | Complementa o código com explicações claras, facilitando o entendimento e manutenção futura.           |
| **Feedback rápido**                 | Projeta o código para que erros sejam detectados rapidamente, agilizando correções.                    |
| **Performance consciente**          | Evita otimizações prematuras que complicam o código; foca primeiro na clareza e depois na performance. |
| **Consistência**                    | Mantém padrões e estilos uniformes, melhorando a legibilidade e colaboração no time.                   |
